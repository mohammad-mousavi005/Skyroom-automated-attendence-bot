# SkyRoom Chat Monitor Bot 🤖

یک ربات مانیتورینگ چت برای **اسکای‌روم** که به‌صورت مهمان وارد اتاق می‌شود، پیام‌های جدید را جمع‌آوری می‌کند و هر ۵۰ پیام را با هوش مصنوعی تحلیل می‌کند.  
اگر تمام آن ۵۰ پیام **فقط شامل اسم و فامیل فارسی** باشند، ربات به‌طور خودکار **اسم شما را در چت ارسال می‌کند**.

## ویژگی‌ها
- ورود خودکار به اسکای‌روم با حالت **مهمان**
- شناسایی پیام‌های جدید با شناسه‌ی یکتا (بدون تکرار)
- ارسال بچ‌های ۵۰ پیامی به **OpenAI** برای تشخیص محتوای اسم و فامیل
- ارسال خودکار اسم شما در چت در صورت تأیید
- تنظیمات کامل در فایل `config.json` (قابل ویرایش بدون نیاز به تغییر کد)
- لاگ‌گیری کامل در فایل `bot.log` و نمایش در کنسول
- پرامپت هوش مصنوعی قابل شخصی‌سازی در فایل تنظیمات

## نیازمندی‌ها
- **Python 3.8+** 
- **Google Chrome** (نسخه‌ی به‌روز)
- **chromedriver.exe** متناسب با نسخه‌ی کروم شما (آموزش دانلود در ادامه)
- کتاب‌خانه‌های پایتون داخل `requirements.txt`

## نصب و راه‌اندازی

### ۱. دریافت پروژه
فایل‌های پروژه را در یک پوشه (مثلاً Desktop) قرار دهید:
- `skyroom_bot.py`
- `config.json`
- `requirements.txt`

### ۲. نصب کتاب‌خانه‌های پایتون

```bash
pip install -r requirements.txt
```

### ۳. دانلود chromedriver.exe

- نسخه‌ی Google Chrome خود را بررسی کنید (آدرس `chrome://settings/help` در مرورگر).
- chromedriver دقیقاً همان نسخه را از یکی از دو منبع زیر دانلود کنید:

| منبع | توضیح |
|------|--------|
| [Chrome for Testing](https://googlechromelabs.github.io/chrome-for-testing/) | منبع رسمی گوگل (نیاز به فیلترشکن دارد) |
| [dreamshao/chromedriver](https://github.com/dreamshao/chromedriver) | جایگزین بدون فیلتر (GitHub) |

- فایل `chromedriver.exe` را **کنار فایل `skyroom_bot.py`** قرار دهید.

## تنظیمات فایل `config.json`

تمام تنظیمات ربات در فایل `config.json` (کنار اسکریپت) قرار دارد.  
کافیست این فایل را با یک ویرایشگر متن باز کنید و مقادیر زیر را مطابق نیاز خود ویرایش کنید:

```json
{
    "SKYROOM_URL": "https://www.skyroom.online/ch/your-room-name",
    "GUEST_NAME": "مهمان",
    "MY_FULL_NAME": "نام و نام خانوادگی شما",
    "OPENAI_API_KEY": "sk-...",
    "OPENAI_BASE_URL": "https://api.openai.com/v1",
    "OPENAI_MODEL": "gpt-3.5-turbo",
    "BATCH_SIZE": 50,
    "CHECK_INTERVAL": 5,
    "WAIT_TIMEOUT": 20,
    "AI_PROMPT_TEMPLATE": "به پیام‌های زیر ... نگاه کن.\n\nفقط YES یا NO پاسخ بده.\n\nلیست پیام‌ها:\n{numbered_msgs}"
}
```

### توضیح فیلدها:

| فیلد | توضیح |
|------|--------|
| `SKYROOM_URL` | 🔗 لینک اتاق اسکای‌روم (حتماً با `https://` شروع شود) |
| `GUEST_NAME` | 👤 اسمی که ربات با آن به‌عنوان مهمان وارد می‌شود |
| `MY_FULL_NAME` | ✏️ اسم و فامیل شما که در صورت تأیید ۵۰ پیام، در چت ارسال می‌شود |
| `OPENAI_API_KEY` | 🔑 کلید API شما (OpenAI-compatible) |
| `OPENAI_BASE_URL` | 🌐 آدرس بیس API (حتماً با `/v1` تمام شود) |
| `OPENAI_MODEL` | 🧠 مدل هوش مصنوعی (پیشنهادی: `deepseek-v4-pro-nothinking` برای پاسخ سریع) |
| `BATCH_SIZE` | 📦 تعداد پیام در هر بچ (پیش‌فرض: ۵۰) |
| `CHECK_INTERVAL` | ⏱️ فاصله‌ی زمانی (ثانیه) بین هر بار بررسی چت |
| `WAIT_TIMEOUT` | ⌛ حداکثر زمان انتظار برای لود شدن عناصر صفحه (ثانیه) |
| `AI_PROMPT_TEMPLATE` | 📝 پرامپت ارسالی به هوش مصنوعی. می‌توانید شخصی‌سازی کنید. متغیرهای `{len_messages}` و `{numbered_msgs}` به‌طور خودکار جایگزین می‌شوند |

## تنظیم API هوش مصنوعی

### روش ۱ – استفاده مستقیم از OpenAI (نیاز به IP غیرایرانی یا VPN)

- یک [API Key](https://platform.openai.com/api-keys) دریافت کنید.
- در `config.json` مقدار `OPENAI_API_KEY` را کلید خود قرار دهید.
- `OPENAI_BASE_URL` را `https://api.openai.com/v1` قرار دهید.
- `OPENAI_MODEL` را مثلاً `gpt-4o-mini` یا `gpt-3.5-turbo` بگذارید.

### روش ۲ – استفاده از پروژه‌ی ds2api (بدون نیاز به فیلترشکن)

با توجه به محدودیت‌های اینترنت ایران، می‌توانید از پروژه‌ی [ds2api](https://github.com/CJackHwang/ds2api/blob/main/README.en.md) استفاده کنید که یک پروکسی محلی برای APIهای OpenAI-compatible است.

1. مخزن ds2api را دریافت و طبق مستندات آن اجرا کنید.
2. پس از اجرا، پروکسی روی آدرس `http://127.0.0.1:5001` فعال می‌شود.
3. در فایل `config.json` تنظیم کنید:
   - `OPENAI_BASE_URL` → `"http://127.0.0.1:5001/v1"`
   - `OPENAI_API_KEY` → یک رشته‌ی دلخواه که خودتان در پروژه ds2api تعیین کردید.
   - `OPENAI_MODEL` → مطابق با مدل نیاز، پیشنهاد برای پاسخ سریع‌تر: `deepseek-v4-pro-nothinking`
4. حالا ربات از طریق پروکسی محلی به API متصل می‌شود.

## اجرای ربات

پس از تکمیل مراحل بالا، در ترمینال (CMD یا PowerShell) دستور زیر را اجرا کنید:

```bash
python skyroom_bot.py
```

- ربات مرورگر را باز کرده، وارد اتاق می‌شود و شروع به مانیتورینگ می‌کند.
- لاگ‌ها هم در کنسول نمایش داده می‌شوند و هم در فایل `bot.log` ذخیره می‌شوند.
- برای توقف، `Ctrl + C` را بزنید.

> 💡 اگر فارسی در ترمینال ناخوانا بود، قبل از اجرا دستور `chcp 65001` را اجرا کنید.

## رفع مشکلات رایج

### ❌ خطای `chromedriver.exe پیدا نشد!`
- مطمئن شوید فایل `chromedriver.exe` را دانلود و در **همان پوشه‌ی `skyroom_bot.py`** قرار داده‌اید.
- نسخه‌ی chromedriver باید **دقیقاً برابر** با نسخه‌ی Google Chrome شما باشد.

### ❌ خطای `This version of ChromeDriver only supports Chrome version ...`
- نسخه‌ی chromedriver خود را با نسخه‌ی کروم تطبیق دهید (به بخش [دانلود chromedriver](#۳-دانلود-chromedriverexe) مراجعه کنید).

### ❌ ربات پیام‌ها را نمی‌خواند یا دکمه‌ها را پیدا نمی‌کند
- احتمال دارد ساختار HTML اسکای‌روم تغییر کرده باشد. لاگ‌ها (فایل `bot.log`) را بررسی کنید.
- در صورت نیاز، سلکتور‌های بخش `get_messages` یا `send_chat_message` در `skyroom_bot.py` را به‌روز کنید.

### ❌ خطای SSL یا عدم اتصال به OpenAI
- اگر از **روش مستقیم** استفاده می‌کنید، از فعال بودن VPN مطمئن شوید.
- با **روش ds2api**، مطمئن شوید پروکسی روی پورت 5001 اجراست و `OPENAI_BASE_URL` دقیقاً `http://127.0.0.1:5001/v1` باشد (حتماً `/v1` در انتها باشد).

### ❌ خطای `config.json پیدا نشد!`
- فایل `config.json` باید **دقیقاً در کنار** `skyroom_bot.py` باشد.
- نام فایل باید `config.json` باشد (نه `config (1).json` یا پسوند دیگر).

## نکات مهم

- فایل `bot.log` حاوی تاریخچه‌ی کامل فعالیت‌های ربات است.
- با تغییر `BATCH_SIZE` می‌توانید تعداد پیام‌های هر بچ را کم یا زیاد کنید.
- پرامپت `AI_PROMPT_TEMPLATE` را می‌توانید بسته به نیاز خود (مثلاً تشخیص الگوهای خاص) شخصی‌سازی کنید. فقط دو placeholder زیر را حفظ کنید:
  - `{len_messages}` → تعداد پیام‌ها
  - `{numbered_msgs}` → لیست شماره‌گذاری‌شده‌ی پیام‌ها
- ربات مجدداً پیام‌های تکراری را آنالیز نمی‌کند (با کمک `data-message-id` یکتا).
- این ابزار صرفاً برای مقاصد آموزشی و اتوماسیون‌های شخصی طراحی شده است. مسئولیت استفاده بر عهده کاربر است.
